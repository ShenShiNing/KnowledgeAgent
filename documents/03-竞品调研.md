## 3. 主要竞品详情

### 3.1 ChatDOC

| 项目 | 说明 |
|------|------|
| **官网** | [chatdoc.com](https://chatdoc.com) |
| **定位** | 商业化的 AI 文档对话平台 |
| **核心特点** | 文档解析能力强，表格分析优秀 |

#### 支持的文件格式
- 文档类：PDF, Word (.doc, .docx)
- 表格类：Excel (.xls, .xlsx), CSV
- 演示类：PowerPoint (.ppt, .pptx)
- 纯文本：TXT, Markdown
- 其他：HTML, EPUB, RTF, Pages, Numbers, Key
- 图片：JPG, PNG（支持 OCR）

#### 优势
- 文档解析质量高，特别是表格解析
- 提供引用来源，答案可信度高
- 界面简洁易用

#### 劣势
- 闭源产品，无法自建部署
- 不支持自定义 AI 模型
- 价格较高（商业化产品）

### 3.2 Dify

| 项目 | 说明 |
|------|------|
| **官网** | [dify.ai](https://dify.ai) |
| **GitHub** | [langgenius/dify](https://github.com/langgenius/dify) |

## 4. 知识 Agent 的差异化定位

### 4.1 核心差异

| 差异点 | 描述 |
|--------|------|
| **文档编辑** | 支持在线编辑文档，竞品大多不支持 |
| **模型灵活切换** | 支持用户配置多个第三方 API Key，随时切换模型 |
| **本地优先** | 开源可本地部署，数据完全可控 |
| **简洁专注** | 专注知识库场景，不过度复杂化功能 |

### 4.2 竞争策略

1. **开源路线**：采用开源策略，建立社区信任
2. **轻量级**：相比 Dify 更轻量，上手门槛更低
3. **文档编辑**：提供文档编辑功能，形成差异化
4. **模型自由**：支持任意兼容 OpenAI API 的模型

## 5. 市场机会分析

### 5.1 用户痛点
- 商业产品价格昂贵，无法自建部署
- 开源产品配置复杂，上手门槛高
- 现有产品缺少文档编辑功能
- 模型选择受限，无法灵活切换

### 5.2 目标用户
| 用户类型 | 需求描述 |
|---------|---------|
| 个人开发者 | 需要本地部署的知识库工具 |
| 小团队 | 需要低成本、易部署的方案 |
| 企业用户 | 需要数据私密、可定制的方案 |
| 研究人员 | 需要管理大量文献资料的助手 |
